# Security Policy

## Reporting a Vulnerability

We take security very seriously. If you discover a security vulnerability in the ValidTrust SDK, please report it to us responsibly.

### How to Report

Please report security vulnerabilities by contacting the maintainers directly. Do **not** open a public GitHub issue for security vulnerabilities.

You can reach the maintainers at: [INSERT SECURITY CONTACT EMAIL HERE]

When reporting a vulnerability, please include:
- A clear description of the issue
- Steps to reproduce the vulnerability
- Potential impact
- Any suggested fixes or mitigations (if you have them)

### What to Expect

We will acknowledge your report within 48 hours and provide a more detailed response within 72 hours indicating the next steps in handling your report. We may ask for additional information or guidance.

### Disclosure Process

After the initial reply to your report, we will work to confirm the vulnerability and assess its impact:
1. We will keep you informed of the progress toward a fix and full announcement.
2. We may ask you to help test the fix.
3. We will credit you for the discovery in the security advisory (unless you prefer to remain anonymous).
