# Label Documentation

This document describes the labels used in this repository for issue categorization.

## Issue Type Labels

- `bug` - Something isn't working as expected
- `feature` - New feature or enhancement request
- `documentation` - Improvements or additions to documentation
- `maintenance` - Refactoring, dependency updates, or other maintenance tasks

## Contribution Labels

- `good first issue` - Good for first-time contributors
- `help wanted` - Issue needs assistance from the community
- `drips-wave` - Issue eligible for Drips Wave contribution
- `bounty-ready` - Issue is ready for bounty program participation

## Component Labels

- `frontend` - Frontend-related issues
- `backend` - Backend-related issues
- `smart-contract` - Smart contract-related issues
- `sdk` - SDK-related issues
- `tests` - Test-related issues

## Priority Labels

- `priority: low` - Low priority issue
- `priority: medium` - Medium priority issue
- `priority: high` - High priority issue

## Difficulty Labels

- `difficulty: beginner` - Beginner-friendly issues
- `difficulty: intermediate` - Intermediate-level issues
- `difficulty: advanced` - Advanced-level issues

## Status Labels

- `needs triage` - Issue needs to be triaged by maintainers
- `in progress` - Issue is being worked on
- `blocked` - Issue is blocked by another issue or dependency
- `on hold` - Issue is temporarily on hold
